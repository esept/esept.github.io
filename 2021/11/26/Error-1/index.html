JavaNativeFoundation:GetGlobalVM:Failed to locate @rpath/libjvm.dylib

发表于 2021-11-26 | 分类于 java

JavaNativeFoundation:GetGlobalVM:Failed to locate @rpath / libjvm.dylib for JNI_GetCreatedJavaVMs().

问题解决

系统: macos_big sur

使用本地JDK实验Swing的窗口方法是,IDEA报错

窗口出现,但其中显示的内容为空

https://github.com/hardwarelayer/tientn-jboard-game/issues/3

https://github.com/Homebrew/homebrew-core/issues/66953

https://blog.csdn.net/baofeidyz/article/details/110506645

以上三个方法的内容都是类似的,在run path中简历libjvm.dylib的快捷方式,其中有一部分人成功解决了问题,因此可以尝试一下

其中有一个解决方法是使用homebrew安装jmeter,但是由于brew的二进制下载已经被禁止了,所以不知道是否可行

我一直没能找到其中的@rpath的位置,但是后来了解到这个问题可能只是出现在部分jdk中

然后使用IDEA安装了新的JDK,重新创建项目,使用新的JDK,问题被成功解决·····

所以到底为什么之前的错误没能解决我也不知道,看网上的评论有人说是big_sur本身的问题,在苹果的网站上也有人提到了这个事情····

https://discussions.apple.com/thread/252036741

只不过使用的是Eclipse,但是如果是因为JDK的原因导致的错误的话,那么使用不同IDE应该也是会出现相同的错误

第一篇报错文章,就先这样吧…
