相关：
参数个数不同/参数类型不同/参数的多类型顺序不同
无关：

数组

数组是一种引用数据类型
数组当中的多个数据，类型必须统一
数组的长度在程序运行期间不能改变

数组的初始化：

在内存中创建一个数组，并向其中赋予一些默认值
动态初始化：指定长度

动态初始化数组的格式：

数据类型[] 数组名称 = new 数据类型[数组长度]

左侧数据类型，代表数组中保存的数据，全是统一的什么类型
【】-> 表示这是一个数组
数组名称-> 数组的名字
<=>
new 代表创建数组的动作
数据类型： 必须和左边的数据类型保持一致
中括号的长度： 数组中保存的数据个数，是一个int数字

静态初始化：指定内容:直接将具体的数据内容进行指定

数据类型[] 数组名称 = new 数据类型[] {元素1,元素2···}

使用静态初始化数组的时候，格式还可以在省略一下：可以省略’new 数据类型[]’

访问数组元素

直接打印数组名称得到的是数组对应的内存地址哈希值

访问数组的格式：

数组名称[索引值]

在使用动态初始化数组的时候，其中的元素都会自动有一个默认值
int：0
float：0.0
字符类型：‘\u0000’
布尔类型：false
如果是引用类型：null

java的内存需要划分成五个部分

栈（stack）:存放的都是方法中的局部变量
局部变量：方法的参数，或是方法{}内部的变量
作用域：一旦超出作用域，立刻从栈内存中消失
堆（heap）: 凡事new出来的东西，都在堆当中
堆内存里面的东西都有一个地址值：16进制
堆内存里面的数据都有默认值
整数：0
浮点数：0.0
字符类型：‘\u0000’
布尔类型：false
如果是引用类型：null
方法区(Method Area):存储.class相关信息，包含方法的信息

本地方法栈(Native Method Stack)
寄存器(pc Resister)

ArrayIndexOutOfBoundsException-数组索引越界异常

原因：索引编号错误
解决：修改为存在的正确索引编号

NullPointerException-空指针异常

只赋值了一个null，没有用new进行创建
原因：忘了new
解决：补上new
