静态static关键字

一旦用了static关键字,那么这样的内容不再属于对象自己,而是属于类的,所以凡是本类的对象,都共享同一份

内存图
[方法区] [栈] [堆]
Student.class{ Student 1 -> new Student 1
String name; Student 2 -> new Student 2
int age; [静态区]
static String room -> room <- Student.room
}

根据类型称访问静态变量的时候,全程[和对象没关系,只和类有关系]

如果没有static关键字，那么必须首先创建对象，然后才能通过对象使用它
无论成员变量，还是成员方法，只要有了static，对推荐使用类名称进行调用
静态方法：类名称.静态变量
静态方法：类名称.静态方法()
静态不能直接访问非静态:因为在内存中[先]有静态内容,[后]有非静态内容
静态方法中不能使用this关键字:this代表当前对象,通过谁调用谁的方法,谁就是当前对象

静态代码块
public class 类名称{
static{
//静态代码块中的内容
}
}
特点: 当第一次用到本类时,静态代码块执行唯一的一次
静态内容总是优先于非静态,所以静态代码块比构造方法先执行
典型用法:用来一次性的对静态变量进行赋值
